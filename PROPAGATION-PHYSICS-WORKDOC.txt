Status

This is a working document for the question whether the elimination-based book can show a
genuine, complete bridge from K4-structure to physics. Not just the speed of light, but all

It is not part of the book text. It is a staging area for hypotheses, constraints, and

Methodological Commitment: Elimination, Not Construction

The newer book (FirstDistinction.lagda.tex) operates entirely by elimination. This is not
a stylistic preference but the foundational method:

- We do not build propagation and show it works.
- We define the space of all candidates, then show only one survives.

Construction asks: "Can we assemble an object with property P?"
Elimination asks: "Given all possible objects, which ones fail to satisfy the structural
constraints, and what remains?"

This distinction governs every design decision in this document:

- A propagation type that admits only one inhabitant by construction is not an elimination
  proof. It is encoding the answer.
- A propagation type that admits many inhabitants, combined with constraints that filter all
  but one, is an elimination proof.
- The older book's PropagationFactor (a singleton type whose only value is 1) is
  constructive. It does not eliminate; it asserts.
- The newer book's arithmetic statement (max-propagation-per-edge = V ∸ deg = 1, closed by
  refl) is a computation, not an elimination either.

The open problem is: define a genuinely wide type of propagation candidates, then show that
K4 structure, locality, and invariance filter it down to exactly one survivor.

Any proposed formalization in this document must be checked against this criterion:

- Does the type admit alternatives that must be ruled out?
- Or does the type already contain only the desired answer?

If the latter, the proof is circular regardless of how many lemmas surround it.

Every constant must be derived eliminatively from K4 structure. If even one is inserted by

PATH 2: PROPAGATION => c
================================================================================

Core Hypothesis

The candidate bridge from pure K4 structure to measurable physics is the propagation rate.

Working claim:

- In the FD system, the discrete propagation rate is forced as 1 step per step.
- This should be understood as c_FD = 1, not yet as meters per second.
- The physical constant c is then not "computed directly" as 299792458 m/s.
- Instead, c emerges as the conversion structure between discrete space-steps and
  discrete time-steps once the continuum scaling is anchored.

What c = 1 Means Here

In FD, c = 1 should mean:

- 1 spatial step = 1 temporal step
- equivalently: delta-x / delta-t = 1 in native FD units
- this is already physical in the sense of natural units
- it is not yet a SI-valued statement

Important Restriction

We must not claim:

- "FD directly computes 299792458 m/s"

We may only claim:

- FD forces c_FD = 1
- a continuum rescaling maps FD units to SI units
- once a single physical anchor is fixed, the space-time conversion is no longer free

Scaling Picture

Discrete layer:

- distance = number of K4 propagation steps
- time = number of update steps
- propagation bound = 1 edge per step, if this can be proved eliminatively

Continuum layer:

- x_phys = a * x_FD
- t_phys = b * t_FD
- c_phys = a / b

Interpretive consequence:

- if c_FD = 1, then the physical speed of light is the ratio of the chosen length scale to
  the chosen time scale
- c_phys is therefore a scale-conversion fact, not an additional independent primitive

What Must Be Proved Formally

The key theorem cannot be an arithmetic identity alone. The proof target is stronger:

- the admissible propagation rate is forced by structure
- rates different from 1 are eliminated
- the result is compatible with K4 invariance and locality

Required proof shape:

1. Define a genuine propagation structure on the newer elimination-based book side.
2. Show that propagation is constrained by K4 adjacency and lattice locality.
3. Show that any admissible rate must be uniform under the relevant K4 invariance.
4. Exclude rates above 1 because they skip local adjacency / cell mediation.
5. Exclude rates below 1 because they fail to realize the immediate neighbor relation as the
   basic propagation unit.
6. Conclude that exactly 1 survives.

Current State In The Newer Book

This path is now complete in the newer book.

The current formal shape is:

- candidate type: any function K4Vertex -> K4Vertex -> Nat
- locality on the diagonal
- full K4 invariance under vertex permutations
- lower bracket on a distinguished reference edge (v0,v1)
- upper bracket on the same reference edge
- transport of both brackets to every off-diagonal edge by invariance
- sandwich argument forcing the unique admissible law to be adjacency itself

This is now a genuine elimination theorem rather than an arithmetic coincidence.

Current State In The Older Material

The older material contributes two ingredients, but neither is yet sufficient:

- a propagation section with PropagationFactor and "causal-unit"
- a CellTree / foldK4 block expressing locality and per-cell invariance

Formal weakness of the old propagation section:

- it builds a singleton type whose only inhabitant already sits at 1
- therefore it does not eliminate alternatives; it encodes the target value

Formal weakness of the old locality section:

- the supposed context-independence statement is too weak / tautological
- it does not yet exclude non-unit propagation in a meaningful way

The Missing Middle Layer

The missing layer between mathematics and physics is not "a better paragraph" but a new
formal structure.

What is missing:

- a type of admissible propagation laws
- a notion of local one-edge transmission between neighboring K4 instances
- a proof-relevant locality condition strong enough to forbid edge skipping
- a proof-relevant invariance condition strong enough to forbid arbitrary edge-dependent
  rates

Without that layer, c = 1 remains a computed coincidence rather than a forced physical
bridge.

Candidate Formal Shapes For Propagation

The next design decision is critical: what kind of object should propagation be?

This should not be decided by convenience. The wrong choice can smuggle in the desired
uniqueness too early and turn the proof constructive again.

Candidate A: scalar rate only

- form: r : Nat
- reading: a single global propagation rate

Advantages:

- simple to state
- easy to compare numerically with the existing max-propagation-per-edge discussion

Disadvantages:

- too weak structurally
- does not itself say what propagates, between which cells, or under which local rule
- forces the proof to talk in metaphors about "faster" or "slower" without a propagation
  object carrying those constraints

Assessment:

- useful as a final corollary
- not strong enough as the primary eliminative object

Candidate B: functional propagation law

- form: step : Cell -> Cell
- reading: each cell propagates to exactly one next cell

Advantages:

- very crisp
- easy to formulate locality and invariance conditions on a single map

Disadvantages:

- probably too strong too early
- K4 adjacency naturally presents several neighboring cells, not a unique one
- risks building uniqueness into the datatype rather than proving it

Assessment:

- may become valid at a later stage if uniqueness is derived
- should not be assumed as the starting ontology of propagation

Candidate C: relation-valued propagation law

- form: Prop : Cell -> Cell -> Set
- reading: Prop c1 c2 means c2 is an admissible immediate propagation target of c1

Advantages:

- proof-relevant and structural
- naturally supports locality: only neighbors may propagate
- naturally supports totality on adjacency: every required neighbor relation is realized
- does not assume uniqueness in advance

Disadvantages:

- slightly heavier to package and reason about
- cardinality and determinacy must be handled separately if needed later

Assessment:

- currently the safest primary candidate
- keeps elimination honest because alternative propagation patterns can be expressed and
  ruled out rather than excluded by definition

Candidate D: neighborhood-valued propagation law

- form: Prop : Cell -> Powerset Cell
- reading: each cell carries a set of admissible immediate targets

Advantages:

- close to the intuitive language of neighborhoods
- easy to express subset and equality conditions

Disadvantages:

- powerset-style encodings must still be realized constructively in Agda
- often collapses back into a relation anyway

Assessment:

- conceptually acceptable
- probably best implemented extensionally as a relation-valued law

Current Best Guess

The safest progression is:

1. Start with relation-valued propagation.
2. Express locality and adjacency-realization at that level.
3. Only ask about uniqueness or single-valuedness afterward.
4. Extract a scalar rate statement only at the end, as a theorem about the surviving law.

This avoids the main danger:

- defining propagation in a way that already presupposes the very uniqueness we want to
  derive

Core Structural Constraints To Try

These are better starting constraints than talking directly about a scalar rate r.

Locality:

- if Prop c1 c2 holds, then c2 must be adjacent to c1

Adjacency realization:

- if c2 is an immediate required neighbor of c1, then Prop c1 c2 must hold

Invariance:

- propagation behavior is preserved under the relevant K4 symmetries / edge equivalences

Non-skipping:

- no admissible propagation target may bypass an intervening local adjacency layer

Determinacy or cardinal control:

- only add this if it can be derived from the previous constraints rather than assumed

Important Correction To The "r < 1" Discussion

The earlier shorthand "rates below 1 are eliminated" should not be treated as primitive
language.

That phrasing is acceptable informally, but the cleaner structural formulation is:

- a valid propagation law must realize the basic adjacency relation
- anything that fails to realize the minimal required adjacency is eliminated

This is stronger and cleaner than speaking of "slower" propagation, which suggests a
metric or timing model that has not yet been formalized.

Provisional Design Rule

Do not start from:

- "there exists a unique step map"

Start from:

- "there exists an admissible propagation relation subject to locality and invariance"

Then test whether the surviving relation collapses to a unique effective unit law.

About S4

Use caution here.

The working intuition is correct:

- physical observables should be invariant under K4 automorphisms
- all edges are equivalent under the full symmetry of K4

But the current newer book does not yet appear to carry the full formal machinery needed
for a strict "unique S4-invariant propagation law" theorem in the strong sense. The
invariance language exists, but the propagation object it should act on is still missing.

So the safe phrasing is:

- build first toward K4-invariant propagation
- sharpen to explicit S4 language only when the action on propagation data is formalized

Why This Matters For The Physics Claim

If successful, this would change the status of the physics discussion.

Instead of:

- K4 yields some numbers that resemble physics

the argument could become:

- K4 forces a discrete causal structure
- that causal structure forces c_FD = 1
- space and time are therefore linked before phenomenology enters
- once scale is anchored, the light-cone conversion into SI units is fixed

That is a much stronger bridge from formal structure to physics than a bare numerical match.

PATH 3: LOOPS => hbar
================================================================================

Core Hypothesis

K4 has minimal non-trivial cycles: triangles (3-cycles). These are not optional features
but forced by the graph structure. Every vertex of K4 participates in exactly two distinct
triangles.

The candidate bridge to hbar:

- a loop in K4 is the smallest possible closed traversal
- closed traversals define the minimal unit of "action" in the discrete system
- just as c emerges from the forced propagation rate, hbar should emerge from the forced
  minimal loop structure

What hbar = 1 Means Here

In FD, if the argument succeeds, hbar_FD = 1 should mean:

- the smallest non-trivial closed cycle has action 1 in native units
- this is the indivisible quantum of action in the discrete system
- it is not yet a SI-valued statement (like 1.054 * 10^-34 J*s)

Current State

This path is now implemented in the newer book as Forced Minimal Action.

The current formal shape is:

- candidate type: any function K4Triangle -> Nat
- K4Triangle represented by the omitted vertex of each face
- K4 invariance under the full permutation action on faces
- lower bracket on one distinguished reference face tri-012
- upper bracket on the same reference face
- transport of both brackets to every face by invariance
- derived non-triviality from the transported lower bracket
- sandwich argument forcing every face value to 1

What Must Be Proved For hbar

The eliminative proof shape, parallel to Path 2:

1. Define a type of admissible action-unit assignments on K4 cycles.
2. Show that any such assignment must respect the triangle structure of K4.
3. Show that K4 symmetry forces uniform assignment across all triangles.
4. Exclude non-minimal actions (values above 1) because they do not correspond to the
   irreducible cycle structure.
5. Exclude sub-minimal actions (values below 1) because they do not correspond to any
   realizable closed traversal.
6. Conclude that exactly hbar_FD = 1 survives.

Open Questions Specific To Path 3

- What exactly is the formal object? A function from cycles to some action type? A weight
  on the cycle space?
- Is the triangle structure already sufficient, or does the full cycle space of K4 matter?
- How does this relate to the existing triangle-loop-order computation in the book?
- Can the "loop is the minimal recursion" intuition be made precise without importing
  concepts from quantum mechanics?

Danger: The Same Trap As Path 2

The older book's approach to propagation was constructive (singleton type). The same danger
exists here: defining an action type with only one inhabitant. The elimination criterion
from the Methodological Commitment section applies with full force.

PATH 4: DISCRETE GEOMETRY => G
================================================================================

Core Hypothesis

K4 carries intrinsic discrete geometry: it has a well-defined notion of curvature derived
from vertex defect or combinatorial Gauss-Bonnet. The gravitational coupling constant G
should emerge from the forced relationship between this discrete curvature and the energy
content of K4 cells.

What G = 1 Means Here

In FD, if the argument succeeds, G_FD = 1 should mean:

- the coupling between discrete curvature and discrete energy density is fixed
- there is no free parameter governing how "strongly" geometry responds to energy
- in native FD units, the Einstein-like coupling is 1

Current State

This path is now implemented in the newer book as Forced Geometric Coupling.

The key correction compared with the earlier draft is structural:

- G is no longer introduced as a naked scalar satisfying g * chi = chi
- instead it lives on the 0-simplices of K4, parallel to c on edges and hbar on faces
- candidate type: any function K4Vertex -> Nat
- K4 invariance under vertex permutations
- lower bracket on the reference vertex v0
- upper bracket on the same reference vertex
- transport of both brackets to every vertex by invariance
- derived non-triviality from the transported lower bracket
- sandwich argument forcing every vertex value to 1

What Must Be Proved For G

The eliminative proof shape:

1. Define a type of admissible geometry-energy couplings on the K4 lattice.
2. Show that discrete curvature is already determined by K4 structure alone.
3. Show that any consistent coupling must respect the Euler characteristic and local
   curvature constraints.
4. Show that K4 symmetry forces a uniform coupling.
5. Eliminate all couplings except the one forced by the structure.

Open Questions Specific To Path 4

- Is the discrete analog of Einstein's equation a well-defined object on K4, or does it
  require a lattice / simplicial complex extension?
- Does G emerge from the same combinatorial data as c and hbar, or does it require
  additional geometric structure (e.g., simplicial volume, deficit angles)?
- How does the Euler characteristic constraint interact with the coupling?
- Can Regge calculus or discrete differential geometry provide the right formal language
  here, or must it be native to the FD framework?

Status Honesty

Path 4 is the least developed of the three constant-derivation paths. The connection
between K4 geometry and gravitational coupling is more conjectural than the propagation or
loop arguments. This is not a reason to skip it, but a reason to be especially strict about
the elimination criterion.

PATH 5: CLOSURE — NO FREE SCALE
================================================================================

Core Hypothesis

Even if Paths 2, 3, and 4 each succeed individually, the physics bridge is incomplete
without a closure argument:

- with only c: the system has two free scales (length and time are linked, but mass and
  energy are free)
- with c and hbar: one free scale remains (e.g., an overall length scale)
- with c, hbar, and G: the triangle closes and all scales are fixed

The closure lemma states:

  Given c_FD = 1, hbar_FD = 1, and G_FD = 1, no free scale parameter remains.
  The Planck length, Planck time, and Planck mass are determined as:

  l_P = sqrt(hbar * G / c^3) = 1 in FD units
  t_P = sqrt(hbar * G / c^5) = 1 in FD units
  m_P = sqrt(hbar * c / G)   = 1 in FD units

In other words: if all three constants are forced to 1 in native units, then the native
unit system IS the Planck system. There is no remaining freedom.

Why This Is The Decisive Path

Without this path:

- each constant derivation is an isolated result
- the system could still be compatible with arbitrary rescalings
- "physics" would be a loose analogy rather than a forced identification

With this path:

- the three constants lock into a rigid triangle
- any rescaling that preserves one constant must violate another
- the FD unit system and the Planck unit system are identified, not analogized

What Must Be Proved

The closure argument requires:

1. A formal statement that c, hbar, and G are the only independent dimensionful constants
   generated by K4 structure.
2. A proof that they jointly eliminate all free scale parameters.
3. A proof that no additional independent constant arises from K4 that could reintroduce
   freedom.

Point 3 is subtle: if K4 structure forces a fourth independent dimensionful quantity, the
closure argument might need revision.

Current State

This path is now implemented in the newer book.

What is already formalized:

- c_FD = 1, hbar_FD = 1, and G_FD = 1 in the native system
- the three Planck-unit identities in FD coordinates
- a normalized scale-choice record with bridge equations for length, time, and mass
- an elimination theorem forcing the only positive normalized scale triple to be (1,1,1)
- an SI rescaling gate identifying where empirical continuum calibration enters

What is not yet formalized:

- the final analytic extraction of the Planck normalization from arbitrary decimal SI data
- general reciprocal/square-root infrastructure at the late rational/real stage

Planck-Scale Identification

Once closure is established, the identification becomes structural rather than postulated:

- a K4 edge corresponds to the Planck length because it is the minimal propagation unit
  in a system where c = hbar = G = 1
- one update step corresponds to the Planck time for the same reason
- this is not inserted by hand — it follows from the closure of the three constants

Unsafe formulation:

- "One K4 edge is the Planck length by definition"

Safe formulation (requires Paths 2-5 complete):

- FD forces c = hbar = G = 1 in native units
- therefore the native unit system is the Planck system
- therefore one edge = l_P and one step = t_P

PHASED STRATEGY
================================================================================

The first-order path construction is now complete. The next strategy is second-order:

Phase A: Minimal admissibility basis

Goal:

- replace any remaining derived clauses by genuinely primitive ones
- prove which current filters are transported consequences and which are irreducible
- prove that the surviving admissibility basis is unique up to definitional equivalence

Deliverable: a theorem that the current reference-simplex architecture is not only

Phase B: Distinction necessity across frameworks

Goal:

- isolate the minimal notion of distinction actually required for measurement and
  observability
- show that yes/no, equal/not-equal, present/absent discrimination is unavoidable
- show that rival frameworks either fail as measurement theories or import that same kernel
  in disguise

Deliverable: a framework-neutral theorem schema stating that serious physics presupposes a
distinction kernel, whether or not it is named D0.

Phase C: Analytic SI normalization

Goal:

- build the reciprocal layer required on the rational side
- build the square-root layer needed for the Planck formulas
- carry arbitrary SI decimal values into the normalized Planck witness internally

Deliverable: the last external analytic stub removed from the discrete-continuum bridge.

False Paths To Avoid

These look productive but are dead ends:

- Computing c numerically and celebrating the match. That is numerology, not derivation.
- Postulating Planck scale and deriving constants backwards. That is assumption, not proof.
- Completing only one path and claiming physics. One constant leaves the system
  underdetermined.
- Inserting any constant by hand and deriving the others. That smuggles in what should be
  forced.

================================================================================
CROSS-PATH CONSTRAINTS
================================================================================

The three constants are not just independently derived — they must be mutually consistent.

Key cross-constraint: if c, hbar, and G are all forced by the same K4 structure, then
there should be algebraic identities between the combinatorial quantities that generate
them. For example:

- the propagation rate (Path 2) and the loop order (Path 3) both arise from K4 adjacency
- the curvature-energy coupling (Path 4) connects to both via the Euler characteristic

Any successful formalization must eventually show that these cross-constraints hold, not
just that each constant can be derived in isolation.

This is also a falsifiability check: if the three paths produce mutually inconsistent
structural requirements on K4, then the physics bridge fails honestly rather than being
forced by fiat.
